Architecture Web

URL — Anatomie

Une URL (Uniform Resource Locator) identifie de manière unique une ressource sur le web. Elle se décompose en plusieurs parties :

URL 
https://www.monsite.fr:443/dossier/page.php?id=5&cat=info#section

│ protocole │   domaine    │port│   chemin   │   paramètres GET  │ ancre│
PartieValeurDescription
Protocolehttps://HTTP sécurisé (SSL/TLS)
Domainewww.monsite.frNom de domaine du serveur
Port:443443 = HTTPS, 80 = HTTP (optionnel si standard)
Chemin/dossier/page.phpEmplacement du fichier
Query string?id=5&cat=infoParamètres GET
Ancre#sectionLien vers une section (côté client uniquement)

Protocole HTTP — Requête / Réponse

HTTP (HyperText Transfer Protocol) est le protocole de communication entre le client (navigateur) et le serveur. C'est un protocole sans état (stateless).

Requête HTTP
GET /page.php?id=1 HTTP/1.1
Host: monsite.fr
Accept: text/html
Cookie: session_id=abc123
Réponse HTTP
HTTP/1.1 200 OK
Content-Type: text/html; charset=UTF-8
Set-Cookie: session_id=abc123

<html>...</html>

Codes de statut HTTP

✅

2xx — Succès

200 OK — Réponse normale
201 Created — Ressource créée

↪️

3xx — Redirection

301 Moved Permanently
302 Found (redirect temporaire)

❌

4xx — Erreur client

400 Bad Request
403 Forbidden / 404 Not Found

💥

5xx — Erreur serveur

500 Internal Server Error
503 Service Unavailable

Tableaux (Arrays)

PHP — Arrays
<?php
// ── Tableau indexé (indices numériques) ──

echo $fruits[0];           // pomme
$fruits[] = "datte";       // ajout à la fin

    "nom"    => "Dupont",
    "prenom" => "Jean",
    "age"    => 25

Boucles

PHP — Boucles
<?php
// ── for : quand on connaît le nombre d'itérations ──
for ($i = 0; $i < 5; $i++) {
    echo $i . " ";  // 0 1 2 3 4
}

// ── while : tant qu'une condition est vraie ──
$compteur = 0;
while ($compteur < 3) {
    echo $compteur;
    $compteur++;   // ne pas oublier l'incrémentation !
}

// ── do...while : s'exécute AU MOINS une fois ──
do {
    echo "Exécuté";
    $compteur++;
} while ($compteur < 3);

// ── foreach : parcours de tableaux ──
$fruits = ["pomme", "banane", "cerise"];
foreach ($fruits as $fruit) {
    echo $fruit;
}

// foreach avec clé => valeur
$personne = ["nom" => "Alice", "age" => 20];
foreach ($personne as $cle => $valeur) {
    echo "$cle : $valeur";
}

// break et continue
for ($i = 0; $i < 10; $i++) {
    if ($i == 3) continue; // saute le 3
    if ($i == 6) break;    // arrête à 6
}
?>
BoucleUtilisationQuand utiliser ?
forCompteur expliciteNombre d'itérations connu
whileCondition de sortieNombre inconnu, condition à vérifier avant
do...whileExécution garantieAu moins une exécution requise
foreachParcours de tableauxItérer sur tous les éléments d'un array

Traitement PHP

traitement.php
<?php
// ─ Récupération des données ─
$nom   = $_POST["nom"];    // méthode POST
$id    = $_GET["id"];     // méthode GET

// ─ Vérifier si le formulaire a été soumis ─
if ($_SERVER["REQUEST_METHOD"] === "POST") {
    // Traitement du formulaire
}

// ─ Vérifier qu'un champ existe et n'est pas vide ─
if (isset($_POST["nom"]) && !empty($_POST["nom"])) {
    $nom = htmlspecialchars($_POST["nom"]);
    // htmlspecialchars() protège contre les injections XSS
}

// ─ Redirection après traitement ─
header("Location: succès.php");
exit();   // toujours après header()
?>

📤 Méthode GET

  • Données visibles dans l'URL
  • Limité ~2000 caractères
  • Peut être mis en favori / partagé
  • Idéal : recherche, filtres, pagination
  • Ne pas utiliser pour données sensibles

📬 Méthode POST

  • Données dans le corps de la requête
  • Pas de limite de taille notable
  • Non mémorisé dans l'URL
  • Idéal : connexion, inscription, paiement
  • À utiliser pour données sensibles
🛡️
Sécurité : Ne JAMAIS utiliser directement les données utilisateur dans du SQL ou du HTML. Toujours utiliser htmlspecialchars() pour l'affichage et les requêtes préparées pour la base de données.

Sessions & Cookies

Sessions

Une session permet de stocker des informations côté serveur, identifiées par un ID unique (PHPSESSID) stocké dans un cookie côté client.

PHP — Sessions
<?php
// ⚠️ session_start() doit être la PREMIÈRE chose du fichier
session_start();

// ─ Écrire en session ─
$_SESSION["utilisateur"] = "Alice";
$_SESSION["role"]         = "admin";
$_SESSION["panier"]       = ["item1", "item2"];

// ─ Lire en session ─
if (isset($_SESSION["utilisateur"])) {
    echo "Connecté : " . $_SESSION["utilisateur"];
}

// ─ Supprimer une variable de session ─
unset($_SESSION["panier"]);

// ─ Détruire toute la session (déconnexion) ─
session_start();
session_unset();   // vide toutes les variables
session_destroy(); // détruit la session

// ─ Vérification connexion (page protégée) ─
if (!isset($_SESSION["utilisateur"])) {
    header("Location: connexion.php");
    exit();
}
?>

Cookies

Un cookie est stocké côté client (navigateur). Il peut persister après la fermeture du navigateur.

PHP — Cookies
<?php
// setcookie(nom, valeur, expiration, chemin, domaine, sécurisé, httponly)

// ─ Créer un cookie (expire dans 30 jours) ─
setcookie("theme", "dark", time() + (30 * 24 * 3600), "/");

// ─ Lire un cookie ─
if (isset($_COOKIE["theme"])) {
    $theme = $_COOKIE["theme"];
}

// ─ Supprimer un cookie (expiration dans le passé) ─
setcookie("theme", "", time() - 3600);
?>

POO — Programmation Orientée Objet

Classe, attributs, méthodes, constructeur

PHP — Classe de base
<?php
class Voiture {
    // ─ Attributs (propriétés) ─
    private string $marque;
    private string $couleur;
    protected int    $vitesse = 0;
    public static int $nbVoitures = 0;  // attribut statique

    // ─ Constructeur ─
    public function __construct(string $marque, string $couleur) {
        $this->marque  = $marque;
        $this->couleur = $couleur;
        Voiture::$nbVoitures++;   // incrémente le compteur statique
    }

    // ─ Getters (accesseurs) ─
    public function getMarque(): string  { return $this->marque; }
    public function getCouleur(): string { return $this->couleur; }
    public function getVitesse(): int    { return $this->vitesse; }

    // ─ Setters (mutateurs) ─
    public function setCouleur(string $couleur): void {
        $this->couleur = $couleur;
    }

    // ─ Méthodes ─
    public function accelerer(int $km): void {
        $this->vitesse += $km;
    }

    public function freiner(): void {
        $this->vitesse = 0;
    }

    public function afficher(): string {
        return "{$this->marque} {$this->couleur} — {$this->vitesse} km/h";
    }

    // ─ Méthode statique ─
    public static function getNbVoitures(): int {
        return self::$nbVoitures;
    }
}

// ─ Utilisation ─
$v1 = new Voiture("Renault", "rouge");
$v1->accelerer(80);
echo $v1->afficher();            // Renault rouge — 80 km/h
echo Voiture::getNbVoitures(); // 1 (méthode statique)
?>

Héritage

PHP — Héritage
<?php
class VoitureElectrique extends Voiture {
    private int $autonomie;

    public function __construct(string $marque, string $couleur, int $autonomie) {
        parent::__construct($marque, $couleur);  // appel du constructeur parent
        $this->autonomie = $autonomie;
    }

    // ─ Surcharge (override) de la méthode parente ─
    public function afficher(): string {
        return parent::afficher() . " | Autonomie: {$this->autonomie}km";
    }

    public function getAutonomie(): int { return $this->autonomie; }
}

$tesla = new VoitureElectrique("Tesla", "blanc", 500);
$tesla->accelerer(100);
echo $tesla->afficher();  // Tesla blanc — 100 km/h | Autonomie: 500km

// ─ instanceof ─
var_dump($tesla instanceof Voiture);          // true
var_dump($tesla instanceof VoitureElectrique); // true
?>

Classes abstraites & Interfaces

PHP — Abstrait & Interface
<?php
// ── Classe abstraite : ne peut pas être instanciée directement ──
abstract class Animal {
    protected string $nom;

    public function __construct(string $nom) { $this->nom = $nom; }

    abstract public function parler(): string;  // DOIT être implémentée

    public function presenter(): string {
        return "Je suis {$this->nom} et je dis : " . $this->parler();
    }
}

class Chien extends Animal {
    public function parler(): string { return "Wouf !"; }
}

// ── Interface ──
interface Serialisable {
    public function toArray(): array;
    public function toJSON(): string;
}

class Produit implements Serialisable {
    public function __construct(private string $nom, private float $prix) {}
    public function toArray(): array  { return ["nom" => $this->nom, "prix" => $this->prix]; }
    public function toJSON(): string  { return json_encode($this->toArray()); }
}
?>
💡
Mémorisez la différence : classe abstraite = partage du code commun + peut avoir des méthodes concrètes. Interface = contrat pur, toutes les méthodes sont abstraites. Une classe peut implémenter plusieurs interfaces mais n'étend qu'une seule classe.
